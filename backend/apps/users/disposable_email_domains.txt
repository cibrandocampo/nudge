# Curated list of well-known disposable / throwaway email domains, blocked
# from self-signup when settings.BLOCK_DISPOSABLE_EMAIL is True (default in
# prod). One domain per line, lower-case. Comments (#) and blank lines are
# ignored.
#
# Curated from the excellent community-maintained list at
# https://github.com/disposable-email-domains/disposable-email-domains
# (CC0-1.0 — public domain). Huge thanks to that project's maintainers
# for keeping it up to date.
#
# That list has ~4500 entries. The bundled file below is the ~80 most
# common providers — enough to stop casual signup abuse without weighing
# the repo down. For full coverage, replace this file with the upstream
# `disposable_email_blocklist.conf` or extend at runtime via
# DISPOSABLE_EMAIL_EXTRA_DOMAINS (comma-separated env var).
#
# Per-instance corporate domains that happen to match (rare) can be
# allow-listed via DISPOSABLE_EMAIL_ALLOW_DOMAINS.
